Guía de Contribución

Cómo contribuir a los proyectos InnerSource en BCP

Introducción

Contribuir a los proyectos InnerSource en BCP es un proceso sencillo diseñado para ser accesible a todos los empleados, independientemente de su equipo o departamento. Esta guía te llevará a través de los pasos para hacer tu primera contribución.

Antes de comenzar: Asegúrate de tener acceso a GitHub Enterprise y Microsoft Teams. Si no lo tienes, por favor contacta a tu jefatura.
Ten en cuenta: Sigue las guías de contribución (CONTRIBUTING.md) del proyecto sobre el cual deseas hacer una contribución, para revisar requisitos específicos de cada proyecto.

Flujo de Trabajo de Contribución

1

Encuentra un Proyecto

Navega por la página de Proyectos para encontrar un proyecto que te interese. Cada proyecto tiene información sobre su propósito, las tecnologías utilizadas y enlaces a su documentación y repositorio.

2

Revisa la Documentación

Lee la documentación del proyecto para entender su arquitectura, estándares de codificación y pautas de contribución. Cada proyecto puede tener requisitos específicos.

3

Crea un Issue en el proyecto

Crea un Issue en el proecto en el cual quieres contribuir a través de GitHub Issues, agrega toda la información de contexto relevante para tu propuesta de contribución

4

Únete al Canal de Teams

Únete al canal de Microsoft Teams para el proyecto para conectarte con el equipo y hacer cualquier pregunta que puedas tener. Este es también el lugar donde puedes hablar con el equipo owner del proyecto acerca de tu propeusta de contribución.

5

Haz un Fork del Repositorio

Una vez aprobada tu propuesta de contribución, crea un fork del repositorio original del proyecto a tu propia cuenta de GitHub. Esto crea una copia en la que puedes trabajar sin afectar el proyecto original:

1. En GitHub, entra al repositorio original (por ejemplo: github.com/org/proyecto-base).
2. Haz clic en el botón "Fork" (arriba a la derecha).
3. Selecciona tu usuario o tu propia organización como destino. 

github.com/tu-usuario/proyecto-base

6

Configurar el remoto del repositorio original

Para mantener sincronizado tu fork con el repositorio original, agrega el remoto “upstream”:

git remote add upstream https://github.com/org/proyecto-base.git
origin → tu fork (tienes permiso de escritura).
upstream → el repositorio original (solo lectura).

Puedes verificarlo:
 
git remote -v

7

Realiza tus Cambios

Nunca trabajes directamente en main. Crea una rama específica para tu cambio o feature:

git checkout -b feature/mejora-performance

Edita el código, agrega archivos, etc.
Por ejemplo: 
git add .
git commit -m "Agrega sección de buenas prácticas en README"

Subir los cambios a tu fork 
git push origin feature/mejora-performance
8

Envía un Pull Request

1. Ve a tu fork en GitHub.

GitHub te sugerirá crear un Pull Request porque detecta que tu rama difiere del original.


2. Selecciona:

Base repository: el repositorio original (org/proyecto-base)
Base branch: generalmente main
Head repository: tu fork (tu-usuario/proyecto-base)
Compare branch: tu rama (feature/mejora-performance)

3. Escribe un título claro y descripción del cambio según los estándares del programa de Software Engineering Excellence (SEE).

4. Envía el Pull Request.

9

Proceso de Revisión

Los mantenedores del proyecto revisarán tu pull request. Pueden sugerir cambios o mejoras. Sé receptivo a los comentarios y realiza los ajustes necesarios.

Pautas de Revisión de Código

Para Contribuidores
  • Sé receptivo a los comentarios y preguntas
  • Explica tu razonamiento para las decisiones de implementación
  • Divide cambios grandes en pull requests más pequeños y manejables
  • Asegúrate de que tu código siga la guía de estilo del proyecto
  • Escribe mensajes de commit claros que expliquen el propósito de cada cambio
Para Revisores
  • Sé respetuoso y constructivo en tus comentarios
  • Enfócate en el código, no en la persona
  • Explica por qué se necesitan los cambios, no solo qué necesita cambiar
  • Reconoce el buen trabajo y las mejoras
  • Responde a los pull requests de manera oportuna

Reconocimiento y Recompensas

Tus contribuciones a los proyectos InnerSource son valoradas y reconocidas en BCP. Aquí te mostramos cómo reconocemos tus esfuerzos:

  • Clasificaciones de Contribuidores: Los contribuyentes activos son destacados en nuestra página de Rankings.
  • Reconocimiento de Habilidades: Las contribuciones se reconocen en tus revisiones de desarrollo profesional.
  • Reconocimiento Comunitario: Reconocimientos regulares en las comunicaciones de toda la empresa.
  • Campeón de InnerSource: Los principales contribuyentes pueden ser invitados a unirse al programa de Campeones de InnerSource.
